/**
 * Практика #2
 Что произойдет, если в некоторых частях case будет отсутствовать break?
 *
 * @author Aleksandra
 *
 */

Когда break отсутствует в блоке case, выполнение кода продолжается до следующего case или default, даже если условие не совпадает. Это называется "провал" (fall-through). Рассмотрим пример:

int i = 0;
int number = 2;

switch (number) {
    case 1:
        i = 1;
        break; // Оператор break здесь завершает выполнение switch
    case 2:
        i = 2; // Эта строка выполнится, так как number равно 2
    case 3:
        i = 3; // Поскольку нет break, эта строка также выполнится
        break; // Завершение switch
    default:
        i = 0; // Если number равно ни 1, ни 2, ни 3
}

Пояснение:
Если переменная number равна 2, выполнится следующее:
Когда программа дойдет до case 2, будет выполнена строка i = 2;.
Затем, поскольку нет оператора break, выполнение перейдет к case 3, и будет также выполнена строка i = 3;.
В результате, переменная i получит значение 3.

Итог:
Если number равен 1: i будет равно 1.
Если number равен 2: i будет равно 3, поскольку отсутствует break между case 2 и case 3.
Если number равен 3: i будет равно 3 (попадет в case 3).
Если number не соответствует ни одному из case, выполнится блок default, и i будет равно 0.
